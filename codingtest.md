# 유형
해시맵 / 문자열 / 빈도수

# 왜 해시맵인가
이름이 문자열이고, 동명이인이 존재 → 단순 비교 불가  
→ 각 이름의 등장 횟수를 O(N)에 관리해야 함

# 핵심 아이디어
참가자 +1, 완주자 -1을 하면  
완주하지 못한 사람만 값이 1로 남음

# 처음 접근 (틀린 이유)
참가자 / 완주자 각각 해시맵 생성 후 비교하려 함

# 왜 틀렸나
두 맵 비교는 추가 순회 필요 → 구현 복잡  
차이만 남기는 구조를 못 만듦
# 공부법
평일: 3문제 풀기(공부한 자료구조/알고리즘을 먼저 백준으로 유형 별 5문제이상 풀어 감을 잡고 프로그래머스에서 문제 풀기),
복습: 평일 루틴에 정답을 봐서 푼 문제를 1문제 씩 복습하면서 풀고 오답 노트와 함께 공부하기,
주말: 실전을 연습하기 위해 90분 타이머 맞추고 3문제를 한꺼번에 풀기.
# 공부팁
모르는 문제: 모르는 문제는 30분~1시간 정도 고민해도 안 풀릴 때가 많을 겁니다. 이때 "해설을 보고 이해했다면, 반드시 다음 날 '복습 1문제' 리스트에 넣어서 내 힘으로 다시 풀어야" 합니다. 눈으로 이해한 건 내 실력이 아닙니다.

주말 세트의 구성: 90분 3문제를 구성할 때, 풀은 문제 또는 안 풀은 문제로 본인이 자신 있는 유형 1개 + 약한 유형 2개 식으로 섞으세요. 시험장에서 당황하는 연습을 미리 해야 합니다.

# 구현
```java
Map<String, Integer> map = new HashMap<>();
for(String name : participant){
    map.put(name, map.getOrDefault(name,0)+1);
}
for(String name : completion){
    map.put(name, map.getOrDefault(name,0)-1);
}
for(Map.Entry<String, Integer> entry : map.entrySet()){
    if(entry.getValue()==1){
        return entry.getKey();
    }
}
